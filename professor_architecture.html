1 — The Swarm: Node Roster

NODE 0 — Semantic Router (The Dean)

The gateway. Governs, never executes.
SUPERVISOR
Takes the problem statement and breaks it into a Dependency DAG. Assigns token budgets, time limits, and API cost caps per node. Monitors all circuit breakers globally. On Critic FAIL: decides whether to issue a MICRO replan (patch one node) or MACRO replan (rewrite entire DAG, increment dag_version). On 3rd failure: trips HITL. On budget exhaustion: activates triage_mode. Never writes code. Never touches data. Only governs.

NODE 1 — Competition Intel Agent

Runs before everything. Builds the intelligence briefing.
RESEARCH · KAGGLE
Runs in two modes. Deep Research (Day 1): Scrapes Kaggle discussion forum, searches past similar competitions and their winning solutions, deeply analyzes the evaluation metric and writes a custom scorer if needed, checks for known data issues and leakage patterns. NODE 2 — Validation Architect

Sets the Metric Contract. This is law for every downstream node.
CRITICAL · KAGGLE
The most underrated component. Defines the validation strategy before any model is trained. Detects time-series → uses TimeSeriesSplit. Detects groups → uses GroupKFold. Detects imbalance → uses StratifiedKFold. Then runs a CV sanity check: trains a simple model and verifies CV score correlates with public LB. If it doesn't — STOPS THE ENTIRE PIPELINE and escalates. Also writes the Metric Contract which defines the exact scorer function, direction, and explicitly forbids all other metrics. This contract is injected into every agent's system prompt.

NODE 3 — Data Engineer Agent

Data plumbing. Autonomous. Zero human intervention.
DATA

NODE 4 — Feature Factory Agent

Continuous hypothesis-driven feature generation with null importance filtering.
FEATURES · KAGGLE
Runs in rounds, guided by competition_brief.json. Round 1: basic transforms. Round 2: domain features from briefing. Round 3: aggregation features (groupby stats). Round 4: CV-safe target encoding. Round 5: embedding features for text/categoricals. Every candidate feature is tested with Null Importance Filtering — shuffled 50 times to build a null distribution. Feature only survives if its real score beats 95th percentile of its own shuffled versions. Statistical significance tested via Wilcoxon before any "this feature helps" decision is made.

NODE 5 — ML Optimizer Agent

Trains a diverse portfolio. All decisions use the Metric Contract.
MODELING

NODE 6 — Red Team Critic Agent

Adversarial by design. Assumes the Optimizer is guilty until proven innocent.
CRITICAL · ADVERSARIAL
The most important node. Does not validate — it attacks. Uses four adversarial vectors:
⚡ Vector 1 — Data Leakage Hunt

Trains model on shuffled targets — if score stays above random, a feature encodes the target. Checks temporal leakage (t+1 predicting t). Trains model on ID column only — non-random performance = ID leakage. Runs adversarial train/test classifier — AUC > 0.6 = distribution shift.

🎭 Vector 2 — Metric Gaming Detection

Audits prediction distribution histogram (healthy = spread, gamed = spikes). Checks calibration curve. Verifies score beats majority-class baseline AND 1-feature logistic baseline. Detects p-hacking: re-runs top 5 Optuna trials with different seeds — unstable = lucky, not learned.

🔎 Vector 3 — Feature Integrity Audit

Permutation importance on holdout (not train) — high train / zero holdout = noise. Cross-references every engineered feature against competition_brief.json domain knowledge. Feature ablation test: removes top-10 features one at a time — collinear = not truly contributing.

💥 Vector 4 — Robustness Stress Test

Injects Gaussian noise into top features — graceful degradation required, collapse = overfit. Slice performance audit on all meaningful subgroups — overall AUC can hide catastrophic failure on a subgroup. OOF calibration check for stacking safety.

CRITIC VERDICT: {"severity": "CRITICAL", "action": "REPLAN", "remove_features": ["rolling_7d"], "rerun_nodes": ["feature_factory", "optimizer"], "do_not_rerun": ["validation_architect"]}
o1-pro — deep adversarial reasoning leakage_detector · noise_injector · slice_auditor critic_verdict.json FAIL → Supervisor rewrites DAG, never retries same plan

NODE 7 — Ensemble Architect Agent

Where top Kaggle finishes are made. Diversity-first, overfit-resistant.
ENSEMBLE · KAGGLE
Layer 1 (Blending): Weighted average of OOF predictions — weights optimized by Optuna with L2 regularization. Layer 2 (Stacking): OOF predictions become features for a meta-learner (Ridge or XGB). Diversity Check first: prunes models with prediction correlation > 0.98 before any weight optimization. Constrained optimization: max parameters = N/100. Penalizes weights < 0.05. Always compares ensemble vs best single model on a holdout that was never used in weight optimization. Validates ensemble improvement with Wilcoxon test.

2 — Learning Noise Defenses

Five distinct noise types, each with a concrete engineering solution. These are enforced by code, not by hope.

NOISE 1

CV Variance Noise — Treating statistical noise as signal

Model A: 0.8812 CV. Model B: 0.8819 CV. Professor picks B. The difference is within noise. On private LB, A wins.
Fix: Wilcoxon Signed-Rank Significance Gate
Compare fold-level scores with wilcoxon(fold_scores_A, fold_scores_B). If p_value > 0.05: NO SIGNIFICANT DIFFERENCE — keep simpler model. Applied before every "this is better" decision in the entire pipeline. Occam's razor enforced by code.
NOISE 2

Feature Selection Noise — Lucky features from multiple comparisons

200 candidate features. ~10 will improve CV by chance alone. These "lucky" features hurt on private LB.
Fix: Null Importance Filtering (50-shuffle test)
For every candidate feature: shuffle it 50 times, train model each time, build null distribution of scores. Feature is only kept if its real score exceeds the 95th percentile of its own null distribution. Kills lucky features before they poison the model.
NOISE 3

HPO Tuning Noise — Lucky random seeds in Optuna

Best Optuna trial scores 0.8847. Re-run it: scores 0.8801. Lucky fold assignment — not real improvement. Professor locks in overfit hyperparameters.
Fix: Multi-Seed Stability Validation
Top 10 Optuna configs re-run with 5 different seeds each. Ranked by stability_score = mean_cv - (1.5 × std_cv). Most stable config wins, not luckiest. Variance is penalized explicitly.
NOISE 4

Ensemble Weight Noise — Overfit blend weights

Optuna finds perfect weights for OOF predictions. On private LB, ensemble does worse than the best single model.
Fix: Constrained Diversity-First Ensemble
Diversity pruning first (drop models with correlation > 0.98). Max weight parameters = N/100 (statistical DOF constraint). L2 regularization on weights. Ensemble validated on holdout never used in weight optimization. Wilcoxon test confirms ensemble beats best single model.
NOISE 5

Forum/Research Signal Noise — Blindly trusting unvalidated claims

Forum post: "Feature X gave me a huge boost!" Professor adds it. CV drops. The poster was wrong, on different data, or lying.
Fix: Hypothesis-First Forum Pipeline
Every forum insight = HYPOTHESIS, never fact. Must pass: (1) null importance filter, (2) Wilcoxon significance test vs baseline. Only then added to the feature set. Professor never blindly implements suggestions.
3 — Circuit Breakers & Escalation Taxonomy

Each failure level triggers a fundamentally different response — not a retry of the same plan.

1
MICRO REPLAN — First failure at any node

Fix only the specific failing node. Same overall DAG. Targeted patch. node_failure_counts[node] increments.

2
MACRO REPLAN — Second failure, or Critic returns CRITICAL verdict

Critic sends failure_report.json to Supervisor. Supervisor rewrites the entire DAG from scratch. All downstream nodes reset. dag_version increments. replan_count increments.

3
HUMAN IN THE LOOP — Third failure

Exact state saved to Redis checkpoint. System pauses. You receive: what was attempted, why it failed each time, 3 suggested manual interventions. You unblock it, system resumes from checkpoint.

4+
COMPETITION TRIAGE MODE — Budget/time exhausted

Abandons failing strategy. Falls back to: LightGBM + minimal feature set + simple StratifiedKFold. A good submission beats no submission. triage_mode = true in state.

4 — Sealed Pitfall Registry
P-01

Infinite Retry Loop

Escalating replan taxonomy. Same failure twice = new strategy, not new attempt.
P-02

Context Window Bloat

Token budget enforced per agent. Agents receive compressed briefing summaries. Old tool results evicted after 3 turns, stored in Redis.
P-03

Metric Misalignment

Metric Contract written by Validation Architect at start. Injected into every agent's system prompt. Forbidden metrics list enforced.
P-04

Stale Research

Daily delta scans patch the master briefing. New forum insights, host clarifications, and LB shakeups are caught automatically.
P-05

Public LB Overfit

Gap trend monitoring. 48-hour submission freeze if gap increases 3 consecutive times. Final submissions: best CV + most different model.
P-06

Agent Role Contamination

Hard allowed/forbidden tool lists per agent. Enforced at the tool layer, not just in the prompt. Violations logged in lineage trail.
P-07

Memory Poisoning

Memory quality scoring: helpfulness_rate, validated_by_critic, confidence threshold. Bad memories decay below retrieval threshold over time.
P-08

Execution Security

E2B Firecracker MicroVMs. Every code execution in isolated sandbox. Spins up in 150ms, destroyed immediately. Agents never touch local machine.
P-09

Data Leakage (Temporal)

Critic Vector 1. Shuffled target training, ID-only model check, adversarial train/test classifier. All features validated strictly backward-looking.
P-10

State Payload Bloat

State schema uses pointers only. All data lives in /mnt/. Agents exchange metadata in strict JSON. Raw data never enters any context window.
P-11

Provenance Loss

Every state mutation gets SHA-256 hash. Immutable lineage log in SQLite. If an agent alters a feature, there is a permanent record of why, when, and by whom.
P-12

Critic Toothlessness

Critic is adversarial by design with 4 attack vectors. Verdict schema is structured. On CRITICAL: Supervisor rewrites DAG. Critic does not retry — it replans.
P-13

Broken CV / LB Mismatch

Validation Architect runs sanity check. If CV doesn't correlate with public LB — pipeline halted entirely. Fix CV before training anything.

6 — Professor's Golden Rules (Hard-Coded)
Rule 1: If CV score and Public LB score are not correlated — STOP. Fix the validation. Do not train another model. A broken validation strategy means every experiment is meaningless.
Rule 2: Never select a more complex model over a simpler one unless the improvement is statistically significant (Wilcoxon p < 0.05). Occam's razor is enforced by code.
Rule 3: Agents pass pointers, never payloads. Raw data never enters any context window. Ever.
Rule 4: The same failure twice is not a retry problem — it is a strategy problem. On second failure, rewrite the entire DAG.
Rule 5: Every forum insight is a hypothesis, not a fact. It must beat null importance AND Wilcoxon before entering the feature set.
Rule 6: Final 2 submissions: (1) best CV score, (2) most different model from submission 1. Never two similar models as finals.
Rule 7: Memory quality decays. Only memories with helpfulness_rate > 0.6, validated_by_critic = true, and confidence > 0.7 are retrieved. The system gets smarter, not noisier.
